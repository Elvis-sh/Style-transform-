# 图像风格迁移实验报告

> **实验题目:** 基于感知损失的快速图像风格迁移
> **风格原型:** `2.jpg` | **训练数据:** COCO val2017 (5000张) | **设备:** NVIDIA GPU (CUDA)
> **日期:** 2026年6月14日

---

## 一、实验目的

图像风格迁移是计算机视觉领域的经典任务：给定一张内容图像和一张风格图像，
生成一张新图像——其语义内容与内容图像一致，而纹理、色彩、笔触等艺术风格与风格图像一致。

本实验采用 Johnson et al. 在 ECCV 2016 提出的
**"Perceptual Losses for Real-Time Style Transfer and Super-Resolution"**
快速风格迁移框架。与传统 Gatys et al. (2015) 方法（每张图需数百次反向传播迭代优化像素值）不同，
该框架训练一个**前馈卷积神经网络 (TransformerNet)**，将风格迁移从迭代优化问题转化为
**单次前向传播**，使推理速度提升 1000 倍以上，达到实时应用水平。

---

## 二、神经网络模型详情

本实验系统由两个核心神经网络组成：(1) **图像变换网络 TransformerNet**——负责执行风格迁移；
(2) **感知损失网络 VGG-16**——负责计算内容损失和风格损失，指导 TransformerNet 训练。

### 2.1 图像变换网络 — TransformerNet

#### 2.1.1 总体架构

TransformerNet 采用 **Encoder–Residual Bottleneck–Decoder** 的对称沙漏结构：

```
输入: 内容图像 x ∈ R^(B×3×H×W)
                         │
        ┌────────────────┴────────────────┐
        │         ENCODER (编码器)          │
        │  目的: 压缩空间维度，扩展通道维度    │
        │                                  │
        │  Block 1: Conv2d 9×9, 3→32      │  ReflectionPad2d(4)
        │           + InstanceNorm2d       │  保持空间尺寸不变
        │           + ReLU                 │  感受野: 9×9
        │                                  │
        │  Block 2: Conv2d 3×3, 32→64     │  stride=2, padding=1
        │           + InstanceNorm2d       │  尺寸: H/2 × W/2
        │           + ReLU                 │
        │                                  │
        │  Block 3: Conv2d 3×3, 64→128    │  stride=2, padding=1
        │           + InstanceNorm2d       │  尺寸: H/4 × W/4
        │           + ReLU                 │
        └────────────────┬────────────────┘
                         │ 特征图: (B, 128, H/4, W/4)
        ┌────────────────┴────────────────┐
        │     RESIDUAL BLOCKS (残差块×5)    │
        │  目的: 学习风格变换的残差映射       │
        │                                  │
        │  每个 ResidualBlock:              │
        │    ┌─ Conv3×3(128→128, bias=False)│
        │    │  + InstanceNorm2d           │
        │    │  + ReLU                     │
        │    │  + Conv3×3(128→128, bias=False)│
        │    │  + InstanceNorm2d           │
        │    └─→ + 输入(恒等映射) ──→ 输出  │
        │                                  │
        │  ×5 次堆叠，通道和尺寸均不变        │
        └────────────────┬────────────────┘
                         │ 特征图: (B, 128, H/4, W/4)
        ┌────────────────┴────────────────┐
        │         DECODER (解码器)          │
        │  目的: 恢复空间分辨率，生成风格化图像 │
        │                                  │
        │  Block 1: Upsample(×2, nearest)  │
        │           + Conv2d 3×3, 128→64  │
        │           + InstanceNorm2d       │  尺寸: H/2 × W/2
        │           + ReLU                 │
        │                                  │
        │  Block 2: Upsample(×2, nearest)  │
        │           + Conv2d 3×3, 64→32   │
        │           + InstanceNorm2d       │  尺寸: H × W
        │           + ReLU                 │
        │                                  │
        │  Block 3: ReflectionPad2d(4)     │
        │           + Conv2d 9×9, 32→3    │
        │           + Tanh                  │  输出范围 [-1, 1]
        └────────────────┬────────────────┘
                         │
输出: 风格化图像 y ∈ [-1, 1]^(B×3×H×W)
      经 deprocess: y' = clamp(y, -1, 1) × 0.5 + 0.5 ∈ [0, 1]
```

### 2.2 感知损失网络 — VGG-16

#### 2.2.1 网络结构与截取策略

VGG-16 是牛津大学 Visual Geometry Group 提出的经典卷积神经网络，
在 ImageNet 1000 类分类任务上预训练。本实验使用 `torchvision.models.vgg16(pretrained=True)`
加载预训练权重，**冻结全部参数**（`requires_grad = False`），仅用作特征提取。

完整的 VGG-16 features 部分包含 31 层（13 个卷积层 + 5 个最大池化层 + 13 个 ReLU），
本实验将其按 ReLU 层切分为 **4 个不重叠的层段 (slices)**：

```
VGG-16 Features 完整结构:
┌─────────────────────────────────────────────────────┐
│ Block 1: Conv2d(3→64, 3×3) → ReLU         [idx 0-1]│
│          Conv2d(64→64, 3×3) → ReLU        [idx 2-3]│  ← slice1 在此结束
│          MaxPool2d(2×2, stride=2)          [idx 4]  │     (relu1_2)
├─────────────────────────────────────────────────────┤
│ Block 2: Conv2d(64→128, 3×3) → ReLU       [idx 5-6]│
│          Conv2d(128→128, 3×3) → ReLU      [idx 7-8]│  ← slice2 在此结束
│          MaxPool2d(2×2, stride=2)          [idx 9]  │     (relu2_2)
├─────────────────────────────────────────────────────┤
│ Block 3: Conv2d(128→256, 3×3) → ReLU     [idx 10-11]│
│          Conv2d(256→256, 3×3) → ReLU     [idx 12-13]│
│          Conv2d(256→256, 3×3) → ReLU     [idx 14-15]│  ← slice3 在此结束
│          MaxPool2d(2×2, stride=2)         [idx 16]  │     (relu3_3)
├─────────────────────────────────────────────────────┤
│ Block 4: Conv2d(256→512, 3×3) → ReLU     [idx 17-18]│
│          Conv2d(512→512, 3×3) → ReLU     [idx 19-20]│
│          Conv2d(512→512, 3×3) → ReLU     [idx 21-22]│  ← slice4 在此结束
│          (后续 Conv5 层不使用)                         │     (relu4_3)
└─────────────────────────────────────────────────────┘
```

**截取映射表：**

| Slice | VGG 原始索引 | 对应的 ReLU 层 | 输出通道数 | 感受野级别 | 在损失中的用途 |
|-------|------------|---------------|-----------|-----------|-------------|
| slice1 | 0–3 (4层) | relu1_2 | 64 | 局部纹理 (3×3) | 风格损失 ✓ |
| slice2 | 4–8 (5层) | relu2_2 | 128 | 简单模式 (5×5) | 风格损失 ✓ |
| slice3 | 9–15 (7层) | relu3_3 | 256 | 语义部件 | **内容损失 + 风格损失** |
| slice4 | 16–22 (7层) | relu4_3 | 512 | 高层语义 | 风格损失 ✓ |

**关键设计决策:** 不使用 conv5 层（VGG 最深层），因为 conv5 的特征图分辨率过低（H/16 × W/16），
会导致内容损失难以精确约束空间结构，同时深层特征的 Gram 矩阵容易引入训练不稳定性。

### 2.3 损失函数详解

#### 2.3.1 Gram 矩阵 — 风格的数学表示

Gram 矩阵是本实验风格损失的核心。对于特征图 F ∈ R^(C×H×W)（C 个通道，每个通道 H×W 个激活值），
其 Gram 矩阵 G ∈ R^(C×C) 定义为：

$$G_{ij} = \sum_{k=1}^{H \times W} F_{ik} \cdot F_{jk}$$

即第 i 个通道和第 j 个通道特征图的内积。等效的矩阵形式：

$$G = \frac{F \cdot F^T}{C \cdot H \cdot W}$$

其中 F 被 reshape 为 (C, H×W)。除以 C·H·W 进行归一化，消除图像尺寸对 Gram 值量级的影响。

**Gram 矩阵的直观含义:**
- Gᵢⱼ 较大的值表示通道 i 和通道 j 的激活模式**共现性高**（例如：竖线纹理检测器
  和蓝色调检测器经常同时激活 → 风格图中蓝色竖线纹理丰富）
- Gᵢⱼ 较小表示两个通道的激活模式**独立**（它们分别检测不相关的视觉模式）
- Gram 矩阵**丢弃了空间位置信息**（通过内积对所有空间位置求和），只保留特征通道间的
  **相关性结构**——这正是 "风格" 的本质：风格的纹理、笔触、色彩搭配不依赖于
  它们在画面中的具体位置

#### 2.3.2 内容损失 (Content Loss)

内容损失衡量生成图像与内容图像在**语义特征空间**中的差异，而非原始像素空间。
使用 VGG-16 的 `relu3_3` 层（slice3）特征：

$$L_{content}(\hat{y}, y_c) = \frac{1}{C_3 H_3 W_3} \sum_{i,j,k} \left( \phi_3(\hat{y})_{i,j,k} - \phi_3(y_c)_{i,j,k} \right)^2$$

即：$$L_{content} = \text{MSE}\left( \phi_{relu3\_3}(\hat{y}), \; \phi_{relu3\_3}(y_c) \right)$$

其中 φ₃(·) 表示 VGG-16 第 3 段（slice3, relu3_3）的输出特征图。
选择 relu3_3 而非更浅或更深层的理由：
- 更浅层（relu1_2/relu2_2）：特征过于底层（边缘、角点），约束太严格会导致生成图像
  像素级复制内容图，丧失风格
- 更深层（relu4_3/relu5_3）：语义过于抽象，空间分辨率过低，内容结构约束不足，
  可能导致内容图像中的物体形状发生扭曲
- relu3_3 是**中层特征**，恰好平衡了语义保真度和风格自由度

#### 2.3.3 风格损失 (Style Loss)

风格损失衡量生成图像与风格图像在**多层 Gram 矩阵**上的差异：

$$L_{style}(\hat{y}, y_s) = \frac{1}{4} \sum_{l=1}^{4} \text{MSE}\left( \text{Gram}(\phi_l(\hat{y})), \; \text{Gram}(\phi_l(y_s)) \right)$$

其中 l ∈ {1,2,3,4} 分别对应 slice1(slice4 (relu1_2, relu2_2, relu3_3, relu4_3)。
**多层的必要性:**
- slice1 (relu1_2)：约束**细粒度纹理**（如笔触的微观方向性、局部色彩过渡）
- slice2 (relu2_2)：约束**中等纹理模式**（如纹理的周期性、图案重复）
- slice3 (relu3_3)：约束**部件级风格**（如物体局部的色彩分布）
- slice4 (relu4_3)：约束**全局风格**（如整体色调、构图风格）

单层匹配只能捕获一个尺度的风格，多层联合约束才能使生成图像在**多个空间尺度上**
都与风格图一致。

#### 2.3.4 总变差损失 (Total Variation Loss)

$$L_{TV}(\hat{y}) = \frac{1}{N} \sum_{i,j} \left[ (\hat{y}_{i,j+1} - \hat{y}_{i,j})^2 + (\hat{y}_{i+1,j} - \hat{y}_{i,j})^2 \right]$$

惩罚相邻像素的剧烈强度变化，起到**空间平滑正则化**作用。在快速风格迁移中，
TV Loss 有效抑制了转置卷积/上采样过程中产生的**棋盘伪影 (Checkerboard Artifacts)**。

#### 2.3.5 总损失

$$L_{total} = \alpha \cdot L_{content} + \beta \cdot L_{style} + \lambda \cdot L_{TV}$$

本实验的超参数设置为：
- α = **1.0** (内容权重)：保持内容结构
- β = **1×10⁵** (风格权重)：强烈施加风格，β/α = 100,000:1 的比例确保风格成为优化的主导方向
- λ = **1×10⁻⁶** (TV 权重)：轻微平滑正则，避免棋盘伪影但不过度平滑纹理

## 三、模型设计亮点分析

### 3.1 从"逐张优化"到"一次前馈"：推理速度的范式变革

**传统方法 (Gatys et al., 2015):**
- 将风格迁移建模为像素空间的迭代优化问题
- 从内容图（或白噪声）出发，对每张新图片执行 300–500 步 L-BFGS/Adam 优化
- 每步需完整前向+反向传播通过 VGG-19
- **每张图耗时 30–300 秒**，无法实时应用

**本实验方法 (Johnson et al., 2016):**
- 将风格迁移建模为**图像到图像的翻译问题**
- 离线训练 TransformerNet（约 5–10 分钟）学习 `内容图 → 风格化图` 的映射
- 训练完成后，推理仅需**一次前向传播**通过 TransformerNet
- **每张图耗时 25–50 毫秒**，提速 600–12,000 倍，达到 **20 FPS** 实时水平

**核心洞察:** 同一个风格可以无限次复用——将风格"固化"到网络权重中，
用前馈计算替代迭代搜索，是算力换效率的经典策略。

### 3.2 Instance Normalization：风格迁移的关键归一化选择

本实验中 TransformerNet 的**所有归一化层均使用 InstanceNorm2d 而非 BatchNorm2d**。
这一设计选择对风格迁移具有决定性影响：

| 归一化方式 | 归一化范围 | 对风格迁移的影响 |
|-----------|----------|---------------|
| BatchNorm | 跨 Batch 维度（N×H×W） | 跨样本统计量混合，抹平单张图片的色彩/对比度特征 |
| **InstanceNorm** | 单样本内跨空间（H×W） | 每张图保持独立统计量，保留个体风格表达能力 |

**Ulyanov et al. (2016)** 的实验证实：在风格迁移任务中，将 BatchNorm 替换为 InstanceNorm
可显著提升生成质量，因为 InstanceNorm 对每张图像的**对比度归一化**恰好与风格迁移中
"让每张图呈现统一风格色调"的目标一致——InstanceNorm 的 affine 参数 (γ, β) 可以学习到
风格图的目标对比度和色彩偏移量。

### 3.3 感知损失：在特征空间而非像素空间度量差异

传统图像到图像翻译任务（如超分辨率、去噪）通常使用 L1/L2 逐像素损失。
但逐像素损失有两个严重缺陷：

1. **模糊效应:** L2 损失的最优解是"所有可能输出的平均值"，导致生成图像趋于模糊
2. **过度约束:** 强制像素值精确匹配，不给纹理/色彩变化留自由度

本实验将损失计算从**像素空间提升到 VGG 特征空间**：
- VGG-16 在 ImageNet 上预训练，其卷积核已经学会了检测有意义的视觉特征
  （边缘→纹理→部件→物体）
- 在 VGG 特征空间中度量差异，更符合人类的**感知相似度**
- 允许像素值有较大偏差，只要高层语义特征保持一致

**本质:** 用另一个深度网络 (VGG-16) 的隐空间距离替代显式的逐像素距离，
是一种 "Learned Metric" 的思路。

### 3.4 残差连接：内容保真度的保障

TransformerNet 的核心是 5 个残差块。残差连接的数学形式：

$$\text{ResBlock}(x) = \mathcal{F}(x) + x$$

其中 F(x) 是两层 Conv+InstanceNorm+ReLU 的复合函数，x 是恒等映射。

**三个关键作用：**

1. **梯度高速公路:** 反向传播时，梯度可以通过恒等映射路径**无损地**传播到编码器层，
   有效避免了深层网络中的梯度消失问题

2. **学习残差而非全量变换:** 模型只需学习"风格引起的特征偏移量"，
   而内容结构通过恒等映射 x 得以保留。这隐式地实现了**内容/风格解耦**：
   内容走 skip connection，风格走 F(x)

3. **训练稳定性:** 残差块的输出方差更可控，配合 InstanceNorm 使得 5 个残差块堆叠后
   信息流不至于退化

### 3.5 大核卷积 + 镜像填充：无边界伪影

- **首层和末层使用 9×9 大卷积核**（而非常见的 3×3）：在空间分辨率最高的位置提供
  更大的感受野，有助于在编码的起点捕获**全局光照和色调信息**，在解码的终点
  平滑地融合风格纹理

- **ReflectionPad2d(4)**（镜像填充 4 个像素）：与普通的 Zero Padding（填 0）
  不同，镜像填充将图像边缘的像素**对称反射**到边界外。这避免了 Zero Padding
  在卷积后产生的**黑色边界伪影**（图像四周出现暗框），使风格化结果的边缘过渡自然

### 3.6 无 Bias 卷积设计

TransformerNet 的所有卷积层均设置 `bias=False`。原因：
- 当卷积层后紧跟 InstanceNorm 时，卷积的 bias 项是**冗余的**——
  InstanceNorm 先减去均值（消除 bias 的平移效果），再除以标准差，然后通过
  自己的 affine 参数 γ,β 重新施加平移和缩放
- 去掉 bias 减少约 0.5% 的参数量，且不影响表达能力

### 3.7 最近邻上采样 (Nearest Neighbor Upsample)

Decoder 使用 `Upsample(mode='nearest')` 而非转置卷积 (ConvTranspose2d)：
- 转置卷积容易产生**棋盘伪影**——因为不同输出位置的感受野重叠不均匀
- 最近邻上采样 + 普通卷积的组合（Resize-Convolution）被证明是
  抑制棋盘伪影的有效手段 (Odena et al., 2016)
- 最近邻插值直接复制像素值，缺乏可学习参数，但后接的 3×3 卷积负责
  对放大后的特征进行平滑和修正

## 四、实验结果 — 损失值分析

总损失函数定义为三项加权和：

$$L_{total} = \alpha \cdot L_{content} + \beta \cdot L_{style} + \lambda \cdot L_{TV}$$

其中 α=1.0, β=1×10⁵, λ=1×10⁻⁶。以下从训练收敛过程、各项损失行为、推理泛化、损失平衡四个维度展开分析。

### 4.2 风格损失 (Style Loss) 专项分析

风格损失刻画生成图像与风格图像在 VGG-16 四层 Gram 矩阵上的平均差异，β=1×10⁵ 的高权重使其成为优化器的优先目标。

**收敛轨迹：**

- **初始阶段 (Batch 1–50)：** Style Loss 均值 4.84×10⁻⁴。Batch 2 出现异常峰值 2.85×10⁻³（约为平均的 6 倍），这是训练早期的典型现象——某批图像与风格图极为不同导致梯度震荡，但下一批即刻回落至正常水平。
- **快速收敛阶段 (Batch 50–200)：** Style Loss 从 3.58×10⁻⁴ 骤降至 5.95×10⁻⁵，降幅 **83%**。这是高风格权重的直接效果——Adam 迅速将梯度指向风格最小化方向。
- **精细收敛阶段 (Batch 200–500)：** 从 5.90×10⁻⁵ 降至 2.70×10⁻⁵，降幅 54%。下降速度明显放缓，进入指数衰减尾部。此时 Gram 矩阵的匹配已非常精确，剩余误差主要来自不同内容图像之间的风格适应。
- **饱和平台 (Batch 500–1000)：** Style Loss 稳定在 2.5–3.0×10⁻⁵ 区间，最小值 2.03×10⁻⁵。表明在给定模型容量和训练数据下，风格损失已达到**实际下限**——进一步压缩的空间极小。

**加权贡献变化（贯穿训练始末）：**

| 阶段 | 原始 Style Loss | 加权贡献 (×10⁵) | 占 Total 比例 |
|------|----------------|-----------------|--------------|
| 初期 (1–50批) | 4.84×10⁻⁴ | 48.4 | ~75%（主导） |
| 中期 (200–250批) | 5.10×10⁻⁵ | 5.1 | ~30% |
| 末期 (900–1000批) | 2.61×10⁻⁵ | 2.6 | ~19.5% |

> 趋势解读：训练初期 Style Loss 是 Total Loss 的**绝对主力**（75%），驱动 TransformerNet 快速建立风格映射。随着训练推进，风格表达的边际收益递减，Style Loss 在 Total Loss 中的贡献从 75% 降至不到 20%，优化重心自然转移到内容保真度上。

### 4.3 内容损失 (Content Loss) 专项分析

内容损失衡量 VGG-16 relu3_3 层生成图像与内容图像的特征差异，α=1.0 使其在训练后期成为 Total Loss 的主要成分。

**波动特性与成因：**

不同于 Style Loss 的单调下降，Content Loss 全程呈现剧烈波动：从 20.43 到 7.37 之间反复振荡，相邻批次间可出现 5–10 的跳跃。原因有三：

1. **内容图像的多样性：** 每批 4 张图随机裁剪自不同照片，复杂场景（城市街景、人群）天然比简单场景（特写物体、纯色背景）产生更高的 Content Loss
2. **风格—内容的内在张力：** 风格损失追求色彩/纹理变换，内容损失追求结构不变，两者存在天然对抗。高风格权重迫使模型在早期优先满足风格约束，暂时牺牲内容保真度
3. **随机裁剪效应：** 256×256 的随机裁剪意味着同一张照片的不同部位可能产生截然不同的 Content Loss（前景物体 vs 背景纹理）

**整体下降趋势验证：**

尽管存在波动，但阶段均值验证了稳定下降趋势：

| 阶段 | Content Loss 均值 | 较前阶段变化 |
|------|------------------|------------|
| 前 50 批 | 16.38 | — |
| 中段 (200–250) | 15.31 | ↓ 6.5% |
| 最后 100 批 | 10.77 | ↓ 34.2% |

Epoch 2 相比 Epoch 1 均值从 15.45 降至 12.03，降幅 22.1%。最终值 11.21 较初始值 20.43 下降 45.1%。

**一个反直觉观察：内容损失最小值 7.37 出现在训练中段（非最终阶段）**

这说明在某批"容易"的内容图像上，模型曾达到更低的 Content Loss，但最终模型在该值之上收敛（11.21）。原因：训练的目标是**泛化到所有图像**的平均性能，而非对单批图像过拟合。最终值 11.21 代表的是模型在所有训练样本上的**稳健平衡点**。

### 4.4 总变差损失 (TV Loss) 专项分析

TV Loss 惩罚相邻像素差异，λ=1×10⁻⁶ 使其在 Total Loss 中的占比可忽略不计（<0.1%），其作用纯粹是平滑正则化。

**TV Loss 的上升趋势（0.078 → 0.121, +55%）：**

这一上升**不是退化信号**，而是风格注入的必然副作用：

- Style Loss 下降 → 模型输出更丰富的纹理和笔触 → 相邻像素出现更多高频变化 → TV Loss 自然升高
- 最小值 0.0011 出现在训练早期（Batch 12），此时风格尚未注入，模型输出接近内容图的模糊版本
- λ=1×10⁻⁶ 确保 TV Loss 在任何情况下都不会主导优化方向，仅作为底线的"防抖"约束

2. **Style Loss (2.89×10⁻⁵) ≈ 训练末期 (2.61×10⁻⁵)：** 差异仅 11%，证明模型对未见照片的风格表达**高度一致**，未出现过拟合——风格已成功"编码"进网络权重而非记忆训练样本。

3. **标准差分析：** Content Loss 标准差 2.01（样本间差异反映了图片复杂度本身），Style Loss 标准差仅 7.17×10⁻⁶（几乎无差异），再次验证风格表达的**跨样本稳定性**。

**极端案例分析：**

- **最优 (#10, 000000521601.jpg)：** Content=4.18, Style=4.43×10⁻⁵。两项指标均为最优/次优——说明"内容保留"和"风格表达"并非严格此消彼长。构图简洁的照片可以同时实现低内容损失和高风格匹配。
- **最弱 (#8, 000000405249.jpg)：** Content=11.66, Style=2.09×10⁻⁵。高 Content Loss + 低 Style Loss 暗示该图可能包含大量细节纹理使内容匹配困难，而模型倾向于优先保持风格一致（Style Loss 反而最低）。

### 4.6 损失平衡与权重贡献分解

将原始损失乘以各自权重，分析在 Total Loss 中的实际贡献：

| 损失项 | 原始值 | 权重 | 加权贡献 | 占 Total 比例 |
|--------|-------|------|---------|-------------|
| Content Loss | 8.51 | 1.0 | **8.51** | **74.6%** |
| Style Loss | 2.89×10⁻⁵ | 1×10⁵ | **2.89** | **25.3%** |
| TV Loss | 0.12 | 1×10⁻⁶ | 0.00 | <0.1% |
| **Total** | — | — | **11.40** | 100% |

**关键发现：**

尽管 Style Weight (=1×10⁵) 是 Content Weight (=1.0) 的 100,000 倍，但加权后 Content 贡献约为 Style 的 3 倍。原因在于 Style Loss 的原始值已被压至 10⁻⁵ 量级——**几乎触及给定模型容量下的下限**。

这一现象揭示了两阶段训练动力学：
- **前期（Content Loss > 15）：** Style Loss 是高权重的受益者，迅速从 4.84×10⁻⁴ 降至 ~5×10⁻⁵，降幅 90%。此时 Style 贡献 > Content 贡献。
- **后期（Content Loss ≈ 10–12）：** Style Loss 进入饱和区（2–3×10⁻⁵），模型将剩余优化容量配置给 Content Loss。Content 贡献从 25% 升至 75%。

**优化建议：** 若需要更强的风格效果，应将 Style Weight 提升至 5×10⁵（当前风格的饱和区会上移）；若需要更好的内容保真度，应将 Style Weight 降至 5×10⁴（释放优化容量给 Content Loss）。

## 五、结论

本实验成功训练了一个基于 TransformerNet + VGG-16 感知损失的快速风格迁移模型，
以 `2.jpg` 为风格原型。主要结论如下：

1. **模型架构合理:** Encoder-Residual-Decoder 结构（1.68M 参数）能有效学习"
   内容→风格化图像"的前馈映射

2. **损失收敛充分:** 经过 1000 batch 训练 (5.7 分钟)，Content Loss 下降 45%，
   Style Loss 下降 94%，总损失下降 47%

3. **风格表达稳定:** 在 10 张未见实景照片上，Style Loss 保持在 2.89×10⁻⁵
   的稳定水平，泛化能力良好

4. **推理速度达标:** 稳态推理时间 25–38 ms/张（30+ FPS），远超实验要求中
   "风格迁移时间不得过长"的标准

5. **内容保留可进一步提升:** 可通过增加训练数据量、延长训练轮数 (3–5 epochs)、
   适度降低 Style Weight (如 5×10⁴) 来降低 Content Loss
